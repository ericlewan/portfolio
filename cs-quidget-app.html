Quidget  ·  2023–2025

Designing a product
from zero to launch

Designing the complete Quidget product — dashboard, AI configuration, testing, and channel management — as a single coherent system that makes running an AI agent feel effortless.

Role

Founding Product Designer

Timeline

2023–2025

Company

Quidget

The problem

When you're building a B2B SaaS product from scratch, there's no existing UI to iterate on. No legacy patterns to work around. No inherited navigation structure. Every screen, every interaction, every component needs to be designed — and they all need to feel like they belong to the same product.

The challenge is that a product's complexity directly reflects its capability. A powerful AI chatbot platform needs to let users configure model behavior, manage knowledge sources, customize branding, test responses, and deploy to multiple channels. Making this feel effortless without hiding capability is the core design problem.

AI Model — selecting knowledge sources, training data, and conversation behavior in one unified view.

The approach

I designed the product as a single coherent system — not screen by screen, but as a design language applied consistently across every surface. Components were designed before screens. A shared library of inputs, cards, toggles, status indicators, and data displays was established first, then assembled into interfaces.

The navigation structure followed the user's mental model: set up your AI → train it → customize it → test it → deploy it. Each section of the product mapped directly to a step in that journey. Users arriving at any screen for the first time could orient themselves immediately — because the structure was intuitive, not inherited from engineering architecture.

This also meant that as new features were added across two years of development, they felt native to the product rather than bolted on. The system had enough flexibility to grow without breaking visual coherence.

Test & Install — live preview of the configured widget alongside the deployment snippet, reducing the gap between "done" and "live".

The challenge

The hardest problem in complex product design is progressive disclosure — deciding what's visible by default versus accessible on demand. Show too much and users feel overwhelmed. Hide too much and power users feel constrained. There's no universal answer, only a series of judgment calls informed by how different users approach the same task.

I solved this with consistent hierarchy: primary actions always immediately visible, secondary configuration accessible via tabs or expandable sections, advanced settings nested behind deliberate interaction. Users who wanted simplicity got simplicity. Users who needed control found it without hunting.

The other challenge was designing for context diversity. A law firm and a coffee shop both use the same product — completely different content, branding, conversation patterns, and goals. The interface needed to work well for both without feeling generic to either.

Channels — connecting the AI agent to web chat, email, WhatsApp, and other surfaces from a single management view.

Web Chat

The most visible output of the platform was the widget itself — the thing that lived on customers' websites. Designing it meant solving a different kind of problem: making a third-party component feel like it belonged.

Most chat widgets look like widgets. They break immersion, signal "outsourced," reduce trust. I designed the Web Chat as a design system first — every visual property variable: colors, fonts, corner radius, avatar. The widget inherited brand identity from the agent setup flow automatically, so customers rarely had to configure how it looked.

The harder challenge was edge cases at infinite brand variation. A logo that looks great on white disappears on dark. An accent color chosen for fintech creates contrast problems for a children's education brand. I built in automatic light/dark switching based on background luminosity and capped saturation for accessibility — the system protected visual quality without requiring the business owner to understand any of it.

When the branding is right and the AI is on-point, users genuinely can't tell it's a third-party widget. That invisibility was the goal.

Web Chat — the end product. Fully customisable per brand, embedded across thousands of websites.

Marketing site

I also designed the marketing website — built on the same component system as the product, so the visual language stayed consistent from first impression to logged-in dashboard.

The result

Quidget launched, found customers, and kept growing — reaching thousands of businesses with the widget embedded across their websites.

The design outcome was a single visual system running through every surface a user ever touched: the marketing site that acquired them, the onboarding that activated them, the dashboard they returned to every day, and the widget their own customers talked to. One language, no seams.

The metric that mattered most was the one hardest to measure: zero mental energy spent on the interface. When the tool disappears and the work is all that's left — that's when it's working.
